# Security Policy

## Reporting a Vulnerability

We take the security of our smart contracts and protocol very seriously. If you discover a security vulnerability, please report it to us immediately.

**DO NOT** create a public GitHub issue for security vulnerabilities.

### How to Report

Please email your findings to **security@cradlemarkets.com**.

Include as much detail as possible:
- A description of the vulnerability.
- Steps to reproduce the issue.
- Potential impact.
- Any relevant code snippets or proof-of-concept (PoC) code.

### Response Timeline

- We will acknowledge receipt of your report within 48 hours.
- We will provide a preliminary assessment within 5 business days.
- We will keep you updated on the progress of the fix.

### Bounty Program

We may offer a bug bounty for critical vulnerabilities that are responsibly disclosed. The amount will be determined based on the severity and impact of the issue.

### Disclosure Policy

- Please do not disclose the vulnerability publicly until we have had a chance to fix it.
- We will credit you for the discovery in our release notes (unless you prefer to remain anonymous).
